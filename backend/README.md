# CatReader Backend

Go + SQLite backend for shared feed content.

## Prerequisites

- Go 1.23 or newer.

## Run

```bash
cd backend
go mod tidy
go run ./cmd/catreader-server
```

The server listens on `:8080` by default and creates `data/catreader.db`.
Before reading environment variables, the server also loads `.env.local` from
the current directory and `backend/.env.local` when started from the repo root.
Real environment variables take precedence over values in `.env.local`.

Useful environment variables:

```bash
CATREADER_ADDR=:8080
CATREADER_DB_PATH=../data/catreader.db
CATREADER_FEEDS_FILE=../src/data/defaultFeeds.js
CATREADER_REFRESH_INTERVAL=10m
CATREADER_FETCH_TIMEOUT=20s
CATREADER_FETCH_CONCURRENCY=5
CATREADER_LLM_BASE_URL=https://api.openai.com/v1
CATREADER_LLM_API_KEY=...
CATREADER_LLM_MODEL=gpt-4.1-mini
CATREADER_FEED_INTRO_REFRESH_INTERVAL=168h
CATREADER_FEED_INTRO_CHECK_INTERVAL=1h
CATREADER_FEED_INTRO_CONCURRENCY=2
```

For local development, keep private LLM credentials in `backend/.env.local`:

```bash
CATREADER_LLM_BASE_URL=https://api.openai.com/v1
CATREADER_LLM_API_KEY=...
CATREADER_LLM_MODEL=gpt-4.1-mini
```

On startup the server:

1. Migrates the SQLite schema.
2. Seeds feeds from `src/data/defaultFeeds.js`.
3. Starts an initial background refresh when the article table is empty.
4. Refreshes feeds every 10 minutes.
5. Generates and persists feed introductions when LLM config is present, then
   checks hourly and refreshes stale introductions weekly.

This gives each Mac a repeatable initialization path: clone the repo, run the
backend, and the local SQLite database is created and hydrated automatically.

## API

```text
GET  /health
GET  /api/stats
GET  /api/feeds
GET  /api/articles?feed_url=<url>&category=<name>&q=<term>&limit=500&offset=0
GET  /api/articles/{id}
POST /api/admin/refresh
POST /api/admin/feed-intros/refresh
POST /api/admin/llm-config
```

The first phase stores feed content globally. User-specific read state and
reading list data will be added in a later phase behind `syncid`.

Feed introductions are shared feed metadata. They are generated by the backend
with server-side LLM credentials and returned in `GET /api/feeds` when present.
In local development, `POST /api/admin/llm-config` can update `.env.local` from
the AskCat settings UI. It only accepts localhost requests.
